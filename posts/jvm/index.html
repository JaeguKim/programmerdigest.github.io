JVM

Java codeлӮҳ м–ҙн”ҢлҰ¬мјҖмқҙм…ҳмқ„ мӢӨн–үн•ҳкё°мң„н•ң лҹ°нғҖмһ„ нҷҳкІҪмқ„ м ңкіөн•ңлӢӨ. Java bytecodeлҘј machine languageлЎң ліҖнҷҳн•ңлӢӨ. JVMмқҖ JRE(Java Run Environment)мқҳ мқјл¶ҖмқҙлӢӨ.

JVM мһ‘лҸҷл°©мӢқ

лЁјм Җ Java codeк°Җ bytecodeлЎң м»ҙнҢҢмқјлҗңлӢӨ. мқҙ л°”мқҙнҠё мҪ”л“ңлҠ” лӢӨлҘё лЁёмӢ м—җм„ң н•ҙм„қлҗңлӢӨ. л°”мқҙнҠёмҪ”л“ңлҠ” мҶҢмҠӨмҪ”л“ңмҷҖ нҳёмҠӨнҠё мӢңмҠӨн…ңмқ„ м—°кІ°н•ҳлҠ” м–ём–ҙмқҙлӢӨ.

JVM Architecture

1. ClassLoader

class нҢҢмқјл“Өмқ„ лЎңл”©н•ҳлҠ”лҚ° мӮ¬мҡ©лҗҳлҠ” мӢңмҠӨн…ңмқҙлӢӨ. Loading,Linking,Initializationмқ„ лӢҙлӢ№н•ңлӢӨ.

2. Method Area

н•„л“ң м •ліҙ, л©”мҶҢл“ң м •ліҙ(л©”мҶҢл“ң мқҙлҰ„, м ‘к·ј м ңм–ҙмһҗ, лҰ¬н„ҙ нғҖмһ…), нғҖмһ… м •ліҙ, мғҒмҲҳ runtime pool, л©”мҶҢл“ң мҪ”л“ңлҘј м ҖмһҘн•ңлӢӨ.

3. Heap

лӘЁл“ к°қмІҙл“Ө, кҙҖл Ёлҗң мқёмҠӨн„ҙмҠӨ ліҖмҲҳ к·ёлҰ¬кі  л°°м—ҙл“Өмқҙ м ҖмһҘлҗңлӢӨ. мқҙ кіөк°„мқҖ мҠӨл Ҳл“ңл“Өк°„м—җ кіөмң лҗңлӢӨ.

4. JVM Language Stack

лЎңм»¬ ліҖмҲҳл“Өмқ„ м ҖмһҘн•ңлӢӨ, мӨ‘к°„ кІ°кіјл“Ө лҳҗн•ң м ҖмһҘн•ңлӢӨ. к°Ғ мҠӨл Ҳл“ңлҠ” мһҗмӢ л§Ңмқҳ JVM stackмқ„ к°Җм§Җкі  мһҲлӢӨ. н•ЁмҲҳк°Җ нҳём¶ңлҗ л•Ңл§ҲлӢӨ мғҲлЎңмҡҙ н”„л Ҳмһ„мқҙ мғқм„ұлҗҳкі  н•ЁмҲҳнҳём¶ңмқҙ лҒқлӮҳл©ҙ мӮӯм ңлҗңлӢӨ.

5. PC Register

нҳ„мһ¬ мӢӨн–үмӨ‘мқё мқёмҠӨнҠёлҹӯм…ҳ мЈјмҶҢлҘј м ҖмһҘн•ңлӢӨ. к°Ғ мҠӨл Ҳл“ңлҠ” лҸ…лҰҪм Ғмқё PC registerлҘј мҶҢмң н•ңлӢӨ.

6. Native Method Stack

лӢӨлҘё м–ём–ҙлЎң мһ‘м„ұлҗң л„ӨмқҙнӢ°лёҢ мҪ”л“ңм—җ лҢҖн•ң мқёмҠӨнҠёлҹӯм…ҳмқҙ м ҖмһҘлҗңлӢӨ.

7. Execution Engine

InterpreterмҷҖ JIT(Just In time) м»ҙнҢҢмқјлҹ¬, GCлЎң кө¬м„ұмқҙ лҗҳлҠ”лҚ° InterpreterлҠ” л°”мқҙнҠёмҪ”л“ңлҘј н•ңмӨ„н•ңмӨ„ н•ҙм„қн•ҙм„ң мӢӨн–үн•ңлӢӨ. JITлҠ” мһҗмЈј мӢӨн–үлҗҳлҠ” byte codeл“Өмқ„ compileн•ҙм„ң native codeлЎң ліҖнҷҳн•ңлӢӨ.

8. Native Method interface

JVMм—җм„ң лҸҷмһ‘мӨ‘мқё Java codeк°Җ лқјмқҙлёҢлҹ¬лҰ¬мҷҖ л„ӨмқҙнӢ°лёҢ м–ҙн”ҢлҰ¬мјҖмқҙм…ҳмқ„ нҳём¶ңн• мҲҳ мһҲлҸ„лЎқ н•ҳлҠ” н”„л Ҳмһ„мӣҢнҒ¬мқҙлӢӨ.

9. Native Method Library

Execution engineм—җ н•„мҡ”н•ң CмҷҖ к°ҷмқҖ л„ӨмқҙнӢ°лёҢ лқјмқҙлёҢлҹ¬лҰ¬лҘј л§җн•ңлӢӨ.

мҪ”л“ң м»ҙнҢҢмқј.мӢӨн–ү кіјм •

  1. Compiler - high languageлЎң мһ‘м„ұлҗң н”„лЎңк·ёлһЁмқ„ л„ӨмқҙнӢ°лёҢ мҪ”л“ңлЎң ліҖнҷҳ
  2. Linker - м—¬лҹ¬ н”„лЎңк·ёлһЁ нҢҢмқјл“Өмқ„ н•Ёк»ҳ л¬¶м–ҙмӨҢ
  3. Loader - secondary storage(HDD)м—җ мһҲлҠ” нҢҢмқјл“Өмқ„ RAMм—җ лЎңл“ңн•Ё.
  4. Execution - OSмҷҖ н”„лЎңм„ём„ңм—җ мқҳн•ҙм„ң мӢӨн–үлҗЁ.

мһҗл°” мҪ”л“ң м»ҙнҢҢмқј.мӢӨн–ү кіјм •

CмҷҖлҠ” лӢ¬лҰ¬ Linking кіјм •мқҙ л°ңмғқн•ҳм§Җ м•ҠлҠ”лӢӨ. JVMмқҙ RAMм—җ мң„м№ҳн•ҙмһҲкі , мӢӨн–үлҸҷм•Ҳ нҒҙлһҳмҠӨлЎңлҚ”к°Җ нҒҙлһҳмҠӨнҢҢмқјл“Өмқ„ RAMмңјлЎң лЎңл“ңн•ңлӢӨ. мқҙ кіјм •м—җм„ң byte мҪ”л“ңк°Җ ліҙм•ҲмғҒ л¬ём ңк°Җ м—ҶлҠ”м§Җ кІҖмҰқлҗңлӢӨ.

к·ёлҰ¬кі  execution engineмқҙ л°”мқҙнҠёмҪ”л“ңлҘј native machine codeлЎң ліҖнҷҳн•ңлӢӨ. мқҙ кіјм •мқҖ л‘җк°Җм§Җ л°©мӢқмңјлЎң л°ңмғқн•ңлӢӨ.

1. JITк°Җ bytecodeлҘј native codeлЎң ліҖнҷҳ

мқҙкІғмқ„ just in time compilingмқҙлқј л¶ҖлҘёлӢӨ. мһҗл°”к°Җ Cм—җ л№„н•ҙ лҠҗлҰ°мқҙмң мқҙкё°лҸ„ н•ҳлӢӨ. н•ңлІҲ мӢӨн–үмӢңнӮЁ мҪ”л“ңлҠ” мәҗмӢңм—җ м ҖмһҘлҗҳм–ҙ л№ лҘҙкІҢ мӢӨн–үмӢңнӮ¬ мҲҳ мһҲлӢӨлҠ” мһҘм җмқҙ мһҲлӢӨ.

JITлҠ” JVMмқҳ мқјл¶Җмқҙл©° bytecodeлҘј н•ҙм„қн•ңлӢӨ.

2. Interpretterк°Җ byte codeлҘј н•ңмӨ„ н•ңмӨ„ мқҪм–ҙм„ң native codeлЎң ліҖнҷҳн•ңлӢӨ.

JITк°Җ лӘЁл“  bytecodeлҘј native codeлЎң ліҖнҷҳн•ҳлҠ” кІғмқҖ л№„мҡ©мқҙ мҶҢлӘЁ л•Ңл¬ём—җ, Interpreterк°Җ byte codeлҘј ліҖнҷҳн•ҳлӢӨк°Җ мқјм • кё°мӨҖмқҙ м§ҖлӮҳл©ҙ JITк°Җ лҸҷмһ‘н•ңлӢӨ.

мһҗл°”к°Җ м»ҙнҢҢмқјм–ём–ҙмқҙл©ҙм„ң мқён„°н”„лҰ¬н„° м–ём–ҙмқё мқҙмң ?

м»ҙнҢҢмқјлҹ¬м–ём–ҙмқё мқҙмң лҠ” мһҗл°” мҪ”л“ңк°Җ л°”мқҙнҠёмҪ”л“ңлЎң ліҖнҷҳлҗҳкё° л•Ңл¬ёмқҙлӢӨ.

мқён„°н”„лҰ¬н„° мқём–ҙмқё мқҙмң лҠ” JIT,Interpreterк°Җ л°”мқҙнҠёмҪ”л“ңлҘј л„ӨмқҙнӢ°лёҢ лЁёмӢ мҪ”л“ңлЎң ліҖнҷҳн•ҳкё° л•Ңл¬ёмқҙлӢӨ.

Javaк°Җ лҠҗлҰ°мқҙмң 

  1. Dynamic Linking : CмҷҖ лӢ¬лҰ¬, linkingмқҙ лҹ°нғҖмһ„м—җ мқјм–ҙлӮңлӢӨ.
  2. Run-time interpreter : л°”мқҙнҠё мҪ”л“ңлҘј л„ӨмқҙнӢ°лёҢ лЁёмӢ  мҪ”л“ңлЎң ліҖнҷҳлҗҳлҠ” кіјм •мқҙ лҹ°нғҖмһ„м—җ мқјм–ҙлӮңлӢӨ.
